Lazyscraper PDF web scraper

This side project is just a simple web scraper which scrapes allitebooks and downloads pdf books for a given query. This project started because I have an insatiable thirst for knowledge and allitebooks has been my #1 source for finding a book for any programming related topic. I'd spend hours downloading books so I decided to automate that function form me, and the lazyScraper was created. Consisting of only a few lines of code the scraper will make a request with a user defined query and will navigate through all the links of the results to download each book related to the query's topic.

The scraping is done using beautifulsoup which is a library for xml and html parsing for python 2 and 3. This project is available here or by clicking on the image above the description button that opened this dialog.

Video Showroom

This was a freelancing project which involved playing an HD video and turning off everytime a door closed. The client made a 45° inclined box with a door that turns on a light when open and on which the viewer sits. Once inside the door closes and the 1'×1' LED lamp turns off and the video starts. A raspberry was used for this project since it can render 1080p HD video and uses a Debian based linux OS which I could use to my advantage.

I decided to create a web interface on which the client could control most aspects of the system such as:

  • Reboot RPi

  • Shutdown RPi

  • Upload videos

  • Delete videos

  • Preview videos

  • Change current video

  • Test lighting

  • Start GPIO polling

  • Review documentation and contact info

For this task I used the web microframework Flask for the back-end routing and a Celery worker to take care of the GPIO polling.

Now, for the server to be accesible as an offline interface I created a wi-fi hotspot on the raspberry pi using iptables, hostapd, and dnsmasq. This enabled the client to access the server by connecting to the pi's wi-fi and navegating to the pi's static ip on his browser.

Also, since the client did not have much knowledge using unix based systems, a startup script was implemented using bash. This script was then moved to the init.rd folder and, through LSB headers, set to start after the Desktop interface has started. The client was then able to connect to the pi using the method descriped on the last paragraph.

As for the elements used to wire the project a relay and limit switch controlled the output of the light and video. The relay isolated the 120V of the power outlet from the 5V raspberry pi output while the snap action limit switch controlled the relay's state. This project is available here or by clicking on the image above the description button that opened this dialog.
